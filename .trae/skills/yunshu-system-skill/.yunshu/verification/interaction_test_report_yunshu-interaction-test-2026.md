## 四、修复方案

### 修复原则
参照 01-init 的交互定义模式，在每个缺失交互点的阶段补充明确的 `AskUserQuestion` 调用定义。

### 具体修复

#### 02-plan SKILL.md
在"高风险操作清单"后补充：
```markdown
**用户确认交互**（Trae IDE）：
调用 `AskUserQuestion`：
- 问题："以下操作存在[风险描述]，是否继续？"
- 选项："确认执行" / "取消操作" / "查看回滚方案"
```

#### 04-accept SKILL.md
在 Step 7 补充：
```markdown
**验收通过确认交互**（Trae IDE）：
调用 `AskUserQuestion`：
- 问题："验收通过。DoD N/N 条通过，代码质量无问题。是否进入交付阶段？"
- 选项："确认，进入交付" / "需要修改"
```

#### 05-deliver SKILL.md
在 Step 3 补充：
```markdown
**交付方式选择交互**（Trae IDE）：
调用 `AskUserQuestion`：
- 问题："实现已完成。你想怎么处理？"
- 选项："本地合并到主分支" / "推送并创建 Pull Request" / "保留当前分支" / "放弃这次工作"
```

#### 06-recover SKILL.md
在 Step 4 补充：
```markdown
**恢复确认交互**（Trae IDE）：
调用 `AskUserQuestion`：
- 问题："从交接文档恢复：[目标/状态/下一步]。确认从这里继续？"
- 选项："确认继续" / "需要调整"
```

#### 09-software-bridge SKILL.md
在"执行命令前必须向用户确认"后补充：
```markdown
**命令确认交互**（Trae IDE）：
调用 `AskUserQuestion`：
- 问题："即将执行：[命令]。是否确认？"
- 选项："确认执行" / "取消" / "查看回滚方案"
```

## 五、结论

**其他阶段**: 存在 5 个高优先级交互缺失 ❌

**建议**: 按修复方案补充各阶段的 AskUserQuestion 调用定义，确保规范与实际行为一致。
