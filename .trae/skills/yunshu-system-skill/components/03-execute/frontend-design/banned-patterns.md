# 禁止模式清单（Banned Patterns）

> 解决P-015业务理解缺失：避免"AI风格"的通用界面
> 触发条件：设计思考阶段 + 代码自检阶段

---

## 绝对禁止（NEVER）

### 1. 禁止字体

以下字体**绝对禁止**使用：

| 字体 | 原因 | 替代建议 |
|------|------|---------|
| **Inter** | 最常见的"AI字体"，过度使用 | Space Grotesk、Manrope、Plus Jakarta Sans |
| **Roboto** | Google 默认字体，缺乏特色 | Outfit、Sora、Work Sans |
| **Arial** | 系统默认，过于普通 | Helvetica Neue（如可用）、Geist |
| **系统默认字体** | 无特色，缺乏辨识度 | 任何 Google Fonts 特色字体 |
| **Space Grotesk** | 也开始过度使用 | Syne、Clash Display、Cabinet Grotesk |

**例外情况**：
- 用户明确要求使用某字体
- 品牌指南强制要求
- 需要与现有系统保持一致

---

### 2. 禁止色彩方案

以下色彩方案**绝对禁止**使用：

| 方案 | 原因 | 替代建议 |
|------|------|---------|
| **紫色渐变 + 白色背景** | 最典型的"AI风格" | 尝试其他渐变色（蓝/粉/橙）或纯色 |
| **蓝色渐变 + 白色背景** | 第二常见的"AI风格" | 尝试深色背景 + 亮色强调 |
| **胆怯的均匀调色板** | 缺乏重点，平淡无奇 | 主导色 + 锐利强调色 |
| **过度饱和的渐变** | 刺眼、不专业 | 柔和渐变或纯色 |

**例外情况**：
- 品牌色就是紫色/蓝色
- 用户明确要求

---

### 3. 禁止布局模式

以下布局模式**绝对禁止**使用：

| 模式 | 原因 | 替代建议 |
|------|------|---------|
| **标准 12 列对称网格** | 可预测、缺乏特色 | 不对称布局、重叠元素 |
| **三栏等宽布局** | 过于常见 | 非对称比例（1:2:1、黄金分割） |
| **居中单列长页面** | 单调 | 多栏混排、侧边元素 |
| **固定导航 + 内容区** | 千篇一律 | 浮动导航、隐藏导航 |
| **卡片等间距排列** | 缺乏节奏感 | 错落排列、不同尺寸卡片 |

**例外情况**：
- 极简风格（Minimal）允许对称和留白
- 功能优先的工具类应用

---

### 4. 禁止设计模式

以下设计模式**绝对禁止**使用：

| 模式 | 原因 | 替代建议 |
|------|------|---------|
| **圆角卡片 + 阴影** | 最泛滥的组件模式 | 直角卡片、无边框设计 |
| **大图标 + 标题 + 描述** | 特征区三板斧 | 图文混排、插画替代 |
| **Hero 大图 + 标题 + CTA** | 落地页套路 | 视频背景、交互式 Hero |
| ** testimonials 三栏** | 社交证明套路 | 客户 Logo 墙、数据展示 |
| **Pricing 三档卡片** | 定价页套路 | 对比表格、计算器 |

---

### 5. 禁止动效模式

以下动效模式**绝对禁止**使用：

| 模式 | 原因 | 替代建议 |
|------|------|---------|
| **淡入淡出（Fade In/Out）** | 最懒的动画 | 滑动、缩放、旋转 |
| **无限滚动加载动画** | 无意义 | 骨架屏、渐进加载 |
| **过度弹跳（Bounce）** | 幼稚 | 弹性缓动（Elastic Ease） |
| **同时所有元素动画** | 混乱 | 交错动画（Stagger） |
| **无意义的装饰动画** | 干扰用户 | 功能性动画（反馈/引导） |

---

## 趋同警告（WARNING）

以下选择**高度警告**，尽量避免：

| 选择 | 原因 | 替代建议 |
|------|------|---------|
| **Tailwind 默认样式** | 容易趋同 | 自定义配置、扩展主题 |
| **Material Design** | Google 风格明显 | 自定义设计系统 |
| **Bootstrap 样式** | 过时、千篇一律 | 自定义 CSS |
| **Dribbble 流行风格** | 很快过时 | 根据上下文创新 |
| **Awwwards 获奖作品模仿** | 缺乏原创性 | 借鉴思路而非复制 |

## 检查清单

设计完成后，对照以下清单检查：

```markdown
## 禁止模式检查

### 字体
- [ ] 未使用 Inter
- [ ] 未使用 Roboto
- [ ] 未使用 Arial
- [ ] 未使用系统默认字体
- [ ] 未使用 Space Grotesk（除非特定场景）

### 色彩
- [ ] 未使用紫色渐变 + 白背景
- [ ] 未使用蓝色渐变 + 白背景
- [ ] 色彩有明确的主导色和强调色
- [ ] 色彩有层次感

### 布局
- [ ] 非标准 12 列对称网格（极简风格除外）
- [ ] 有打破常规的布局元素
- [ ] 空间构图有特点

### 设计模式
- [ ] 非圆角卡片 + 阴影套路
- [ ] 非 Hero 大图 + 标题 + CTA 套路
- [ ] 有独特的设计元素

### 动效
- [ ] 非简单淡入淡出
- [ ] 动画有目的性
- [ ] 有交错或编排的动画

### 总体
- [ ] 与"AI风格"明显不同
- [ ] 有独特记忆点
- [ ] 风格统一不混乱
```

## 与 03-execute 的集成

### 触发时机
1. **设计思考阶段** — 选择方向时参考禁止清单
2. **代码自检阶段** — 检查是否违反禁止项
3. **设计验证阶段** — 验证是否避免了禁止模式

### 门禁规则
- 违反绝对禁止项 → 阻塞，必须修复
- 触发趋同警告 → 非阻塞，但需说明理由
