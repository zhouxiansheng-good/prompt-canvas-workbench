# 实施计划：Frontend-Design Skill 融合

## 任务分解（WBS）

### 任务1：创建 frontend-design 目录结构
- **输入**：融合方案文档
- **输出**：`frontend-design/` 目录
- **依赖**：无
- **风险**：低

### 任务2：创建核心文件
- **2.1** `README.md` — 融合说明
- **2.2** `design-thinking.md` — 设计思考流程
- **2.3** `aesthetic-checklist.md` — 美学合规检查清单
- **2.4** `banned-patterns.md` — 禁止模式清单
- **输入**：融合方案文档 + Anthropic skill 内容
- **输出**：4个核心文件
- **依赖**：任务1
- **风险**：低

### 任务3：创建设计模板
- **3.1** `design-templates/minimal.md` — 极简风格
- **3.2** `design-templates/maximalist.md` — 极繁风格
- **3.3** `design-templates/retro-futuristic.md` — 复古未来
- **3.4** `design-templates/brutalist.md` — 野兽派
- **输入**：Anthropic skill 美学方向
- **输出**：4个风格模板（先创建4个，后续可扩展）
- **依赖**：任务1
- **风险**：低

### 任务4：修改 03-execute/SKILL.md
- **4.1** Step 2.0 增加设计方向审视（维度6）
- **4.2** Step 2.1 增加前端声明意图模板
- **4.3** Step 2.2 增加设计测试和优先实现
- **4.4** Step 2.4 增加美学合规检查
- **输入**：现有 SKILL.md + 融合方案
- **输出**：修改后的 SKILL.md
- **依赖**：任务2
- **风险**：中（需确保不破坏现有流程）

### 任务5：修改 gates.md
- **5.1** 增加前端设计门禁条目
- **输入**：现有 gates.md + 融合方案
- **输出**：修改后的 gates.md
- **依赖**：任务2
- **风险**：低

### 任务6：验证与自检
- **6.1** 文件清单核对
- **6.2** 目录结构验证
- **6.3** 内容一致性检查
- **输入**：所有产物
- **输出**：验证证据
- **依赖**：任务4、5
- **风险**：低

## 执行顺序

```
任务1（创建目录）
  ↓
任务2（核心文件）+ 任务3（设计模板）【并行】
  ↓
任务4（修改SKILL.md）+ 任务5（修改gates.md）【并行】
  ↓
任务6（验证）
```

## 风险评估

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 修改 SKILL.md 破坏现有流程 | 中 | 高 | 修改前备份，修改后对比验证 |
| 文件内容不完整 | 低 | 中 | 对照融合方案逐项检查 |
| 目录权限问题 | 低 | 低 | 提前确认路径可写 |

## 回滚方案

如需回滚：
1. 恢复 `03-execute/SKILL.md` 备份
2. 恢复 `gates.md` 备份
3. 删除 `frontend-design/` 目录
