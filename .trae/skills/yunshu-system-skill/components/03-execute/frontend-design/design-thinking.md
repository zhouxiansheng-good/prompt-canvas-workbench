# 设计思考流程（Design Thinking）

> 解决P-015业务理解缺失：前端任务的设计维度
> 触发条件：03-execute 识别为前端/UI 任务

---

## 流程概览

```
开始前端任务
  ↓
1. 目的分析（Purpose）
  ↓
2. 调性选择（Tone）
  ↓
3. 约束确认（Constraints）
  ↓
4. 差异化定义（Differentiation）
  ↓
5. 产出设计决策文档（design-decision.md）
```

---

## 1. 目的分析（Purpose）

### 1.1 功能目标
- 这个界面解决什么问题？
- 核心功能是什么？
- 用户的主要操作路径？

### 1.2 目标用户
- 谁使用这个界面？
- 用户的技术水平如何？
- 用户的使用频率？

### 1.3 使用场景
- 在什么设备上使用？（桌面/平板/手机）
- 在什么环境下使用？（办公室/户外/夜间）
- 用户的注意力状态？（专注/碎片化）

---

## 2. 调性选择（Tone）

### 2.1 美学方向选择

必须从以下方向中**明确选择一种**：

| 方向 | 特点 | 适用场景 | 示例参考 |
|------|------|---------|---------|
| **极简（Minimal）** | 极度简洁，留白为主，克制 | 工具类、专业应用、SaaS | Apple、Notion |
| **极繁（Maximalist）** | 丰富、混乱、充满活力，大胆 | 创意类、展示类、艺术 | Gucci、Museum websites |
| **复古未来（Retro-futuristic）** | 80年代科幻感，霓虹、网格 | 科技类、游戏类、音乐 | Synthwave、Cyberpunk 2077 |
| **有机自然（Organic）** | 自然元素、柔和曲线、温暖 | 生活类、健康类、环保 | Aesop、Patagonia |
| **奢华精致（Luxury）** | 高端、精细、优雅、细节 | 金融类、高端服务、珠宝 | Rolex、Cartier |
| **野兽派（Brutalist）** | 原始、粗野、不加修饰、 raw | 艺术类、独立品牌、实验 | Brutalist Websites |
| **装饰艺术（Art Deco）** | 几何图案、对称美、金色 | 时尚类、设计类、酒店 | The Great Gatsby |
| **柔和粉彩（Soft/Pastel）** | 柔和色彩、轻盈感、温柔 | 社交类、女性向、儿童 | Instagram、Pinterest |
| **工业实用（Industrial）** | 功能主义、机械感、金属 | 工具类、B2B、制造 | IBM、Siemens |
| **杂志编辑（Editorial）** | 排版驱动、内容优先、大气 | 内容类、媒体类、博客 | The New York Times、Medium |
| **Playful** | 玩具感、趣味性强、活泼 | 教育类、儿童、游戏 | Duolingo、Nintendo |

### 2.2 选择原则

- **根据目标用户选择**：年轻人 → 极繁/Playful；专业人士 → 极简/工业
- **根据使用场景选择**：夜间使用 → 深色主题；户外使用 → 高对比度
- **避免"安全"选择**：不要每次都选极简
- **避免趋同**：与竞品/常见设计区分
- **大胆选择**：极简和极繁都有效，关键是意图性

### 2.3 调性关键词

选择方向后，提炼 3-5 个关键词：
```
示例（复古未来）：
- 霓虹
- 网格
- 赛博朋克
- 80年代
- 科幻
```

## 3. 约束确认（Constraints）

### 3.1 技术框架
- [ ] React / Vue / Angular / 原生 HTML
- [ ] CSS 框架（Tailwind / Bootstrap / 自定义）
- [ ] 动画库（Framer Motion / GSAP / CSS-only）

### 3.2 性能要求
- [ ] 首屏加载时间 < ? 秒
- [ ] 动画帧率 > ? FPS
- [ ] 包大小 < ? KB

### 3.3 可访问性
- [ ] WCAG 2.1 等级（A / AA / AAA）
- [ ] 键盘导航支持
- [ ] 屏幕阅读器支持

### 3.4 响应式
- [ ] 桌面端（1920px / 1440px / 1280px）
- [ ] 平板端（768px）
- [ ] 移动端（375px）

---

## 4. 差异化定义（Differentiation）

### 4.1 核心记忆点
- 什么让这令人难忘？
- 用户会记住什么？
- 与竞品最大的区别？

### 4.2 独特设计元素
- 特殊的交互方式？
- 独特的视觉元素？
- 创新的布局？

### 4.3 情感目标
- 用户使用后应该有什么感受？
- 信任？愉悦？惊喜？专业？

---

## 5. 设计决策文档

将以上思考整理为 `design-decision.md`：

```markdown
# 设计决策文档

## 任务信息
- **任务ID**：{task_id}
- **功能**：{功能描述}
- **日期**：{YYYY-MM-DD}

## 设计方向
- **美学方向**：{选择的风格}
- **调性关键词**：{3-5个关键词}
- **参考风格**：{具体参考网站/应用}

## 设计决策

### 排版
- **展示字体**：{字体名称}（原因：{为什么选这个}）
- **正文字体**：{字体名称}（原因：{为什么选这个}）
- **字号体系**：
  - 标题：{size}
  - 正文：{size}
  - 辅助：{size}

### 色彩
- **主色调**：{颜色}（原因：{为什么选这个}）
- **强调色**：{颜色}（原因：{为什么选这个}）
- **背景色**：{颜色}
- **文字色**：{颜色}
- **CSS变量定义**：
  ```css
  --color-primary: {值};
  --color-accent: {值};
  --color-background: {值};
  --color-text: {值};
  ```

### 动效
- **页面加载**：{描述}
- **滚动效果**：{描述}
- **悬停状态**：{描述}
- **动画库**：{选择}

### 空间构图
- **布局特点**：{不对称/重叠/对角线等}
- **网格系统**：{描述}
- **负空间使用**：{描述}

### 背景与细节
- **背景效果**：{渐变/纹理/图案等}
- **装饰元素**：{边框/阴影/光标等}

## 差异化元素
- **核心记忆点**：{什么让人难忘}
- **独特设计**：{与常见的区别}

## 禁止项
- [ ] 不使用 Inter/Roboto/Arial/系统字体
- [ ] 不使用紫色渐变+白背景
- [ ] 不使用可预测布局
- [ ] 不趋同于常见选择

## 验证标准
- [ ] 设计是否符合选定美学方向？
- [ ] 是否有独特记忆点？
- [ ] 是否避免了禁止项？
```

---

## 关键原则

1. **设计阶段不可跳过** — 即使是简单组件也要走设计思考
2. **方向必须明确** — 不能模糊地说"好看就行"
3. **文档必须产出** — 没有 design-decision.md = 未完成设计思考
4. **约束必须确认** — 技术/性能/可访问性要提前明确
5. **差异化必须有** — 每个设计都要有独特记忆点
