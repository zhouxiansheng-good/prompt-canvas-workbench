## 门禁1：理解与可持续性门禁

> 解决P-053理解债/P-069长期存活率低

**触发条件**：代码质量审查通过后自动触发

### A. 理解验证（P-053）

1. 读取 `safeguards/cognition.md`
2. 识别关键代码段（核心逻辑/复杂算法/安全相关/边界处理）
3. 生成理解验证问题（每个关键段1-2个问题，覆盖5种类型：为什么这样写/数据流向/边界情况/副作用/修改影响）
4. 评估理解深度：
   - 🟢 深度理解 → 继续可持续性评估
   - 🟡 表面理解 → 补充理解后重新验证
   - 🔴 理解缺失 → 必须重新阅读代码

### B. 可持续性评估（P-069）

1. 读取 `safeguards/sustainability.md`，执行可持续性评估
2. 按6个维度逐项评分（可理解性/可修改性/可测试性/可恢复性/可维护性/可演进性）
3. 计算可持续性评分：
   - 🟢 ≥2.5 → 可持续（预计存活率>60%）
   - 🟡 2.0~2.5 → 需改进（预计存活率30-60%）
   - 🔴 <2.0 → 不可持续（预计存活率<30%），必须修复P0维度后再交付

**关键原则**：理解验证未通过 = 验收未通过；可持续性评分🔴 = 验收未通过

## 门禁2：债务与团队门禁

> 解决P-051技术债务隐蔽/P-052责任链断裂/P-072影子AI/P-073审查瓶颈/P-074安全隐患/P-075~077人才梯队

**触发条件**：理解与可持续性门禁通过后自动触发

### A. 技术债务可见化（P-051）

1. 读取 `safeguards/vibe-coding.md`，执行技术债务检测
2. 按7类技术债务检测清单扫描（假数据/空实现/静默吞错/测试假阳性/过度简化/临时方案/缺失验证）
3. 生成技术债务报告，按🔴/🟠/🟡分级
4. 🔴关键债务必须修复后才能交付

### B. 责任链明确（P-052）

1. 读取 `safeguards/vibe-coding.md`，执行责任链检查
2. 检查责任链：代码审查签名+AI生成标记+决策归属+回滚责任人

### C. 团队AI使用规范（P-072/P-073/P-074）

1. 读取 `safeguards/team-norms.md`
2. 安全审查检查清单（输入验证/权限/SQL注入/XSS/敏感数据/日志安全/依赖安全）
3. 审查加速策略：分批审查（每批≤5文件）+ 优先级审查（安全>核心>辅助）+ 自动化前置

### D. 人才梯队保护（P-075~077）

1. 读取 `safeguards/talent-pipeline.md`
2. 简单任务优先分配初级开发者 + 代码复述理解验证 + 知识传承记录

**关键原则**：🔴关键债务未修复 = 验收未通过；无代码审查签名 = 验收未通过；AI代码不经审查不许合入
