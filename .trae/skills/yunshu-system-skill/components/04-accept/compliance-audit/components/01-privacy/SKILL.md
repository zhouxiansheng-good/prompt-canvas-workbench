# privacy-compliance — 隐私合规审查组件

> 审查网站项目的隐私政策完整性与数据处理合规性。参考 PPChecker 原理，覆盖数据收集、用户权利、第三方披露等核心维度。

---

## 职责

- 检查隐私政策文档是否存在且内容完整
- 检查数据收集范围是否超出必要最小化原则
- 检查用户权利保障机制（访问、删除、更正、撤回同意）
- 检查第三方 SDK / 数据共享披露是否充分
- 检查 Cookie / 追踪技术合规性

---

## 操作流程

### 1. 隐私政策文档检查

1. 扫描项目根目录及常见路径查找隐私政策文件：
   - `privacy-policy.html`
   - `privacy.html`
   - `privacy.md`
   - `src/pages/privacy.*`
   - `public/privacy.*`
2. 若未找到，记录 🔴 致命缺陷：未提供隐私政策
3. 若找到，读取内容并检查以下必要条款是否存在：
   - [ ] 信息收集范围（明确列出收集的数据类型）
   - [ ] 信息使用目的（每个目的需对应具体业务场景）
   - [ ] 信息共享/披露（第三方名称、共享目的、数据类型）
   - [ ] 信息存储与保护（存储地点、期限、安全措施）
   - [ ] 用户权利（访问、更正、删除、撤回同意、注销账号）
   - [ ] 未成年人保护（14周岁以下需监护人同意）
   - [ ] 联系方式（数据保护负责人/邮箱）
   - [ ] 政策更新机制（更新日期、生效日期、通知方式）

### 2. 数据收集范围审查

1. 扫描代码中的数据收集点：
   - 表单字段（`input`, `textarea`, `select`）
   - API 请求体中的字段
   - 本地存储（`localStorage`, `sessionStorage`, `IndexedDB`）
   - Cookie 设置点
2. 对照隐私政策声明，检查：
   - 实际收集的数据是否超出声明范围
   - 是否存在敏感个人信息收集（身份证号、人脸、指纹、定位等）
   - 是否遵循最小必要原则
3. 对AI平台额外检查：
   - 用户输入的提示词是否被记录和存储
   - 生成内容是否关联用户身份
   - 模型训练是否使用用户数据（需明确告知并获取单独同意）

### 3. 用户权利保障审查

1. 检查界面是否提供以下功能入口：
   - [ ] 查看个人数据（数据可携）
   - [ ] 修改/更正个人数据
   - [ ] 删除个人数据（账号注销/数据删除）
   - [ ] 撤回同意（关闭特定功能/退订）
2. 检查后端是否实现对应接口：
   - 数据导出接口
   - 数据删除接口（需级联删除关联数据）
   - 同意撤回接口
3. 检查响应时效：
   - 个保法要求：删除请求应在15个工作日内响应

### 4. 第三方 SDK / 数据共享披露审查

1. 扫描项目依赖中的第三方服务：
   - 分析 `package.json` / `requirements.txt` 中的 SDK
   - 检查代码中引用的第三方 API（统计、广告、支付、CDN）
2. 对照隐私政策，检查：
   - 每个第三方是否都被列出
   - 共享的数据类型是否准确
   - 共享目的是否明确
3. 常见第三方清单：
   - 统计：Google Analytics、百度统计、友盟
   - 广告：Google AdSense、穿山甲
   - 支付：Stripe、支付宝、微信支付
   - 云服务：AWS、阿里云、Cloudflare
   - AI服务：OpenAI、Anthropic、百度文心、阿里通义

### 5. Cookie / 追踪合规性审查

1. 检查 Cookie 同意机制：
   - [ ] 首次访问是否弹出 Cookie 同意横幅
   - [ ] 是否提供"拒绝非必要 Cookie"选项
   - [ ] 是否分类说明必要/分析/营销 Cookie
2. 检查代码中的 Cookie 设置：
   - 是否在获取同意前设置了非必要 Cookie
   - 第三方追踪脚本是否在同意后才加载
3. 检查 Do Not Track 支持：
   - 是否响应浏览器的 DNT 信号

## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_root` | string | 是 | 项目根目录绝对路径 |
| `privacy_policy_path` | string | 否 | 隐私政策文件路径（自动探测） |
| `is_ai_platform` | boolean | 否 | 是否为AI生成内容平台（默认false） |

---

## 输出

| 状态 | 含义 | 后续动作 |
|------|------|----------|
| `PRIV_OK` | 隐私合规审查通过 | 继续下一组件 |
| `PRIV_WARN` | 存在中低危问题 | 记录风险清单，继续审查 |
| `PRIV_FAIL` | 存在致命/高危问题 | 记录风险清单，建议阻断 |

## 风险项模板

| 编号 | 严重度 | 问题 | 法规依据 | 修复建议 |
|------|--------|------|----------|----------|
| P001 | 🔴 | 未提供隐私政策 | 个保法第17条 | 制定并公示隐私政策 |
| P002 | 🔴 | 收集敏感个人信息未单独告知 | 个保法第29条 | 对敏感信息单独告知并获取明示同意 |
| P003 | 🟠 | 隐私政策缺少用户权利条款 | 个保法第44-47条 | 补充访问、更正、删除、撤回同意条款 |
| P004 | 🟠 | 未披露第三方SDK | 个保法第23条 | 列出所有第三方及共享数据类型 |
| P005 | 🟡 | 数据收集超出声明范围 | 个保法第6条 | 删除超范围收集或更新隐私政策 |
| P006 | 🟡 | 未提供账号注销功能 | 个保法第47条 | 提供注销入口及数据删除机制 |
| P007 | 🟡 | Cookie同意机制不完整 | 个保法第13条 | 实现分类同意及拒绝选项 |
| P008 | 🟢 | 隐私政策更新日期超过1年 | 最佳实践 | 审查并更新隐私政策 |
