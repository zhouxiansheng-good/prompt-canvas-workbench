# content-compliance — 内容合规审查组件

> 审查网站项目的内容合规性，重点关注AI生成内容标识、版权归属、UGC责任及禁止内容过滤机制。

---

## 职责

- 检查AI生成内容是否按规定标识（深度合成规定）
- 检查版权归属声明是否清晰
- 检查用户生成内容(UGC)责任条款
- 检查禁止内容过滤机制
- 检查提示词/模板版权风险

---

## 操作流程

### 1. AI生成内容标识审查

1. 判断项目是否涉及AI生成内容：
   - 是否调用大模型API（OpenAI、Claude、文心等）
   - 是否提供AI生成图片/视频/音频功能
   - 是否使用深度合成技术
2. 检查标识要求：
   - [ ] 生成内容是否带有"AI生成""深度合成"等显著标识
   - [ ] 图片/视频是否包含不可擦除的水印标识
   - [ ] 标识位置是否显著，用户无需额外操作即可看到
   - [ ] 是否提供内容溯源信息（生成时间、模型版本）
3. 检查用户端披露：
   - [ ] 服务协议中是否明确告知内容由AI生成
   - [ ] 是否提示用户不要生成违法违规内容
   - [ ] 是否提供投诉举报入口

### 2. 版权归属声明审查

1. 检查网站是否包含版权声明：
   - [ ] 网站底部或关于页面是否有版权声明
   - [ ] 是否声明AI生成内容的版权归属（平台/用户/共有）
   - [ ] 是否声明用户输入提示词的版权归属
2. 检查用户协议中的版权条款：
   - 用户上传内容的版权是否仍归用户所有
   - 平台是否获得必要的使用授权
   - AI生成内容的版权归属是否明确
3. 检查第三方素材使用：
   - 网站使用的图片、字体、图标是否有合法授权
   - 是否使用开源素材但未遵守许可证要求

### 3. UGC责任条款审查

1. 检查用户协议中是否包含：
   - [ ] 用户内容不得违反法律法规的声明
   - [ ] 用户对其发布内容承担全部责任的条款
   - [ ] 平台有权删除违规内容的条款
   - [ ] 用户举报机制说明
2. 检查内容审核机制：
   - [ ] 是否有关键词过滤系统
   - [ ] 是否有AI辅助审核（如内容安全API）
   - [ ] 是否有人工审核流程
   - [ ] 审核响应时效（接到举报后多久处理）
3. 检查避风港条款：
   - 是否提供侵权投诉渠道
   - 是否声明"通知-删除"规则

### 4. 禁止内容过滤机制审查

1. 检查是否建立以下内容的过滤机制：
   - [ ] 危害国家安全、泄露国家秘密
   - [ ] 煽动民族仇恨、民族歧视
   - [ ] 宣扬邪教、迷信
   - [ ] 散布谣言、扰乱社会秩序
   - [ ] 淫秽、色情、赌博、暴力
   - [ ] 侮辱、诽谤他人
   - [ ] 侵犯知识产权
2. 检查技术实现：
   - 输入端过滤（提示词拦截）
   - 输出端过滤（生成内容审核）
   - 图像内容审核（涉黄、涉暴、涉政）
3. 检查日志记录：
   - 是否记录被拦截的内容及原因
   - 是否记录用户违规次数（用于封号决策）

### 5. 提示词/模板版权风险审查

1. 检查平台提供的提示词模板：
   - [ ] 模板是否原创或已获得授权
   - [ ] 是否声明模板的使用许可范围
   - [ ] 用户修改后的模板版权归属
2. 检查用户提示词处理：
   - [ ] 是否将用户提示词用于模型训练（需单独同意）
   - [ ] 是否保留用户提示词副本及期限
   - [ ] 提示词是否与其他用户隔离
3. 检查输出内容版权：
   - [ ] 生成内容是否可能侵犯第三方版权（如生成与知名角色相似的内容）
   - [ ] 是否提供版权风险提示

## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_root` | string | 是 | 项目根目录绝对路径 |
| `terms_of_service_path` | string | 否 | 用户协议文件路径（自动探测） |
| `has_ugc` | boolean | 否 | 是否包含用户生成内容（默认false） |
| `has_ai_generation` | boolean | 否 | 是否包含AI生成功能（默认false） |

---

## 输出

| 状态 | 含义 | 后续动作 |
|------|------|----------|
| `CONT_OK` | 内容合规审查通过 | 继续下一组件 |
| `CONT_WARN` | 存在中低危问题 | 记录风险清单，继续审查 |
| `CONT_FAIL` | 存在致命/高危问题 | 记录风险清单，建议阻断 |

## 风险项模板

| 编号 | 严重度 | 问题 | 法规依据 | 修复建议 |
|------|--------|------|----------|----------|
| C001 | 🔴 | AI生成内容未标识 | 深度合成规定第16条 | 添加"AI生成"显著标识及水印 |
| C002 | 🔴 | 未建立禁止内容过滤机制 | 网络安全法第12条 | 建立输入/输出双层内容过滤 |
| C003 | 🟠 | 用户协议缺少UGC责任条款 | 民法典第1194-1197条 | 补充用户内容责任及平台免责条款 |
| C004 | 🟠 | 未提供侵权投诉渠道 | 信息网络传播权保护条例第14条 | 添加"通知-删除"投诉入口 |
| C005 | 🟡 | AI生成内容版权归属不明确 | 著作权法第11条 | 在用户协议中明确版权归属 |
| C006 | 🟡 | 未记录内容审核日志 | 深度合成规定第7条 | 建立审核日志系统，保留6个月 |
| C007 | 🟢 | 缺少用户举报入口 | 最佳实践 | 在内容旁添加举报按钮 |
| C008 | 🟢 | 提示词模板未声明授权 | 最佳实践 | 添加模板使用许可说明 |
