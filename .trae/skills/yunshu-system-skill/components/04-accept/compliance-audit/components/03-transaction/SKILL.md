# transaction-compliance — 交易合规审查组件

> 审查网站项目的交易合规性，覆盖付费服务条款、退款政策、价格展示、支付安全及税务合规。

---

## 职责

- 检查付费服务条款是否完整、公平
- 检查退款政策是否符合法规要求
- 检查价格展示是否合规（明码标价、无隐藏费用）
- 检查支付安全（PCI DSS 合规性）
- 检查税务合规（发票、税率、纳税义务）

---

## 操作流程

### 1. 付费服务条款审查

1. 检查是否提供服务协议/购买协议：
   - [ ] 服务内容的明确描述
   - [ ] 服务期限（订阅周期、到期处理）
   - [ ] 价格及计费方式
   - [ ] 自动续费条款（是否明确告知、如何取消）
   - [ ] 服务变更/中断的告知义务
   - [ ] 违约责任
2. 检查格式条款公平性：
   - 是否存在免除己方责任、加重对方责任的条款
   - 是否存在排除对方主要权利的条款
   - 重要条款是否以显著方式提示
3. 检查AI平台特有条款：
   - 服务可用性承诺（SLA）
   - 生成内容质量免责声明
   - 使用量限制（Token/次数配额）
   - 超额计费规则

### 2. 退款政策审查

1. 检查退款政策是否存在且合理：
   - [ ] 退款条件（未使用、质量问题、服务不可用）
   - [ ] 退款期限（如7天无理由）
   - [ ] 退款方式（原路返回/账户余额）
   - [ ] 退款处理时效
   - [ ] 不可退款情形（已使用服务、数字商品）
2. 对照法规要求：
   - 中国：网络购买商品七日无理由退货（部分商品除外）
   - 欧盟：14天冷静期
   - 检查是否违反强制性退款规定
3. 检查退款流程：
   - 是否提供自助退款入口
   - 退款申请是否需要不合理门槛
   - 退款失败时的申诉渠道

### 3. 价格展示合规性审查

1. 检查价格展示是否清晰：
   - [ ] 是否显示最终支付价格（含税/不含税需明确）
   - [ ] 是否存在隐藏费用（手续费、服务费）
   - [ ] 货币单位是否明确
   - [ ] 促销价格是否标注原价及折扣依据
2. 检查订阅定价：
   - [ ] 月付/年付价格是否都显示
   - [ ] 年付是否有"相当于每月X元"的换算
   - [ ] 免费试用期结束后是否自动扣费（需明确告知）
3. 检查价格变动：
   - [ ] 调价是否提前通知用户
   - [ ] 老用户是否享有价格保护

### 4. 支付安全审查

1. 检查支付页面安全：
   - [ ] 支付页面是否为 HTTPS
   - [ ] 是否使用可信支付网关（Stripe、支付宝、微信支付）
   - [ ] 是否直接处理信用卡信息（涉及 PCI DSS 合规）
2. PCI DSS 合规检查：
   - 若使用第三方支付（Stripe Checkout、支付宝）：平台不直接处理卡号，PCI 责任由支付商承担
   - 若自建支付表单：需满足 PCI DSS SAQ A-EP 或更高等级
   - 检查是否存储 CVV、磁道数据（禁止存储）
3. 检查支付数据保护：
   - 交易日志是否脱敏（不记录完整卡号）
   - 支付凭证是否加密存储
   - 是否有支付欺诈检测机制

### 5. 税务合规审查

1. 检查发票机制：
   - [ ] 是否支持开具发票
   - [ ] 发票类型（电子/纸质）
   - [ ] 发票申请流程及时效
2. 检查税率处理：
   - [ ] 价格是否明确标注含税/不含税
   - [ ] 是否按用户所在地适用正确税率
   - [ ] 跨境交易是否处理 VAT/GST
3. 检查纳税义务：
   - 平台是否完成税务登记
   - 收入是否如实申报
   - 跨境支付是否代扣代缴

## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_root` | string | 是 | 项目根目录绝对路径 |
| `has_payment` | boolean | 否 | 是否包含付费功能（默认false） |
| `payment_provider` | string | 否 | 支付服务商（stripe/alipay/wechat等） |
| `target_markets` | array | 否 | 目标市场（["CN","EU","US"]） |

---

## 输出

| 状态 | 含义 | 后续动作 |
|------|------|----------|
| `TRANS_OK` | 交易合规审查通过 | 继续下一组件 |
| `TRANS_WARN` | 存在中低危问题 | 记录风险清单，继续审查 |
| `TRANS_FAIL` | 存在致命/高危问题 | 记录风险清单，建议阻断 |

## 风险项模板

| 编号 | 严重度 | 问题 | 法规依据 | 修复建议 |
|------|--------|------|----------|----------|
| T001 | 🔴 | 未提供付费服务协议 | 电子商务法第49条 | 制定并公示服务购买协议 |
| T002 | 🔴 | 未明示退款政策 | 消费者权益保护法第25条 | 明确退款条件、期限、方式 |
| T003 | 🟠 | 价格展示含隐藏费用 | 价格法第13条 | 显示最终支付价格，列明所有费用 |
| T004 | 🟠 | 自动续费未明确告知 | 电子商务法第19条 | 显著提示自动续费及取消方式 |
| T005 | 🟡 | 支付页面非HTTPS | PCI DSS 要求 | 全站启用HTTPS |
| T006 | 🟡 | 未支持发票开具 | 税收征管法第21条 | 提供发票申请入口 |
| T007 | 🟢 | 未提供自助退款入口 | 最佳实践 | 添加退款申请功能 |
| T008 | 🟢 | 未标注价格含税状态 | 最佳实践 | 明确标注"含税"或"不含税" |
