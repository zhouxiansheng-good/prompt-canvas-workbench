# 中国AI与数据法规指南

> 合规审查的法规依据速查手册。涵盖AI生成内容、个人信息保护、数据安全、网络安全等核心法规的关键条款。

---

## 1. 《生成式人工智能服务管理暂行办法》

> 国家互联网信息办公室等七部门，2023年8月15日施行

### 适用范围
- 面向中国境内公众提供生成文本、图片、音频、视频等内容的服务
- 不适用于未向境内公众提供服务的研发活动

### 核心合规要求

| 条款 | 要求 | 审查要点 |
|------|------|----------|
| 第4条 | 遵守法律、尊重社会公德 | 内容不得违法违规 |
| 第7条 | 数据标注合规 | 标注规则、质量评估、人员培训 |
| 第9条 | 服务协议明确 | 告知用户AI生成内容、使用限制 |
| 第12条 | 内容真实准确 | 防止生成虚假信息 |
| 第14条 | 防止沉迷 | 对未成年人采取防沉迷措施 |
| 第15条 | 投诉举报机制 | 提供投诉举报入口 |
| 第16条 | 暂停/终止服务 | 发现违法内容可暂停服务 |

### 算法备案
- 具有舆论属性或社会动员能力的生成式AI服务，需履行算法备案
- 备案主体：服务提供者
- 备案平台：互联网信息服务算法备案系统

## 2. 《互联网信息服务深度合成管理规定》

> 国家互联网信息办公室等三部门，2023年1月10日施行

### 核心合规要求

| 条款 | 要求 | 审查要点 |
|------|------|----------|
| 第6条 | 信息安全主体责任 | 建立健全管理制度和技术保障 |
| 第7条 | 数据和技术管理方案 | 留存日志不少于6个月 |
| 第9条 | 生物识别信息编辑需单独告知 | 人脸/声纹编辑需显著提示 |
| 第12条 | 生成内容标识 | 可能导致混淆的，需显著标识 |
| 第14条 | 非真实身份信息需标识 | 使用非真实身份信息需标识 |
| 第16条 | 深度合成服务显著标识 | 生成内容需添加显著标识 |
| 第17条 | 不得删除、篡改标识 | 标识需不可擦除 |

### 深度合成标识要求
- **显式标识**：用户可直接感知的标识（水印、文字说明）
- **隐式标识**：用户无法直接感知的技术标识（数字水印、元数据）
- 图片/视频：需同时包含显式和隐式标识
- 文本：显式标识（"AI生成""深度合成"）

## 3. 《个人信息保护法》

> 2021年11月1日施行

### 核心合规要求

| 条款 | 要求 | 审查要点 |
|------|------|----------|
| 第6条 | 最小必要原则 | 处理个人信息需有明确合理目的，限于最小范围 |
| 第13条 | 告知-同意规则 | 处理前需告知并取得同意 |
| 第17条 | 告知内容 | 处理者名称、联系方式、目的、方式、种类、保存期限 |
| 第23条 | 向第三方提供信息 | 需告知接收方名称、联系方式、处理目的、方式、种类 |
| 第28条 | 敏感个人信息定义 | 生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹、14周岁以下 |
| 第29条 | 敏感信息处理 | 需具有特定目的和充分必要性，取得单独同意 |
| 第44条 | 知情权 | 个人有权查阅、复制其个人信息 |
| 第45条 | 更正权 | 个人有权请求更正不准确信息 |
| 第47条 | 删除权 | 特定情形下个人有权请求删除 |
| 第51条 | 安全保障义务 | 制定内部管理制度和操作规程，采取加密、去标识化等安全措施 |

### 关键合规红线
- 未经同意收集个人信息 → 违法
- 收集超出必要范围的个人信息 → 违法
- 未提供删除/注销功能 → 违法
- 向第三方提供信息未告知 → 违法
- 处理敏感信息未取得单独同意 → 违法

## 4. 《数据安全法》

> 2021年9月1日施行

### 核心合规要求

| 条款 | 要求 | 审查要点 |
|------|------|----------|
| 第21条 | 数据分类分级保护 | 建立数据分类分级保护制度 |
| 第27条 | 全流程数据安全管理制度 | 数据收集、存储、使用、加工、传输、提供、公开全流程安全 |
| 第29条 | 风险监测与应急处置 | 发现安全缺陷、漏洞立即补救 |
| 第30条 | 重要数据处理者风险评估 | 处理重要数据需明确数据安全负责人 |
| 第31条 | 向境外提供数据 | 重要数据出境需安全评估 |

### 数据分类
- **国家核心数据**：关系国家安全的数据，实行更严格管理
- **重要数据**：一旦泄露可能危害国家安全、公共利益的数据
- **一般数据**：其他数据

## 5. 《网络安全法》

> 2017年6月1日施行

### 核心合规要求

| 条款 | 要求 | 审查要点 |
|------|------|----------|
| 第12条 | 禁止行为 | 不得危害网络安全，不得传播违法信息 |
| 第21条 | 网络安全等级保护 | 制定内部安全管理制度和操作规程，采取防范计算机病毒和网络攻击、网络侵入等危害网络安全行为的技术措施，采取监测、记录网络运行状态、网络安全事件的技术措施，留存相关网络日志不少于六个月 |
| 第22条 | 产品和服务安全 | 网络产品、服务应符合相关国家标准的强制性要求 |
| 第24条 | 实名制 | 网络运营者为用户办理网络接入、域名注册服务，或者为用户提供信息发布、即时通讯等服务，应当要求用户提供真实身份信息 |
| 第40-44条 | 个人信息保护 | 网络运营者收集、使用个人信息应遵循合法、正当、必要原则 |

### 等级保护（等保2.0）
- 第二级：指导保护级（一般系统）
- 第三级：监督保护级（重要系统）
- 第四级：强制保护级（非常重要系统）
- 第五级：专控保护级（极端重要系统）

> 2022年3月1日施行

### 核心合规要求

| 条款 | 要求 | 审查要点 |
|------|------|----------|
| 第7条 | 算法安全主体责任 | 建立健全算法机制机理审核、数据安全等管理制度 |
| 第16条 | 算法推荐服务标识 | 显著标识算法推荐服务 |
| 第17条 | 用户选择退出 | 提供关闭算法推荐服务的选项 |
| 第21条 | 算法备案 | 具有舆论属性或社会动员能力的算法推荐服务需备案 |

## 7. 法规适用速查表

| 场景 | 适用法规 | 核心条款 |
|------|----------|----------|
| AI生成内容未标识 | 深度合成规定第16条 | 显著标识 |
| 未提供隐私政策 | 个保法第17条 | 告知义务 |
| 收集敏感信息未单独同意 | 个保法第29条 | 单独同意 |
| 未启用HTTPS | 网络安全法第21条 | 技术措施 |
| 日志保留不足6个月 | 网络安全法第21条 | 日志留存 |
| 未建立内容过滤 | 网络安全法第12条 | 禁止传播违法信息 |
| 未提供退款政策 | 消费者权益保护法第25条 | 七天无理由 |
| 自动续费未告知 | 电子商务法第19条 | 显著提示 |
| 闭源项目用AGPL | AGPL第13条 | 开源义务 |
| 未保留版权声明 | MIT/Apache许可证 | 版权声明 |

## 8. 合规审查清单

### AI平台必查项
- [ ] 生成内容是否显著标识"AI生成"
- [ ] 图片/视频是否添加不可擦除水印
- [ ] 是否提供算法备案（如适用）
- [ ] 是否建立内容审核机制
- [ ] 用户提示词是否用于训练（需单独同意）
- [ ] 是否提供投诉举报入口
- [ ] 是否采取未成年人防沉迷措施

### 数据合规必查项
- [ ] 是否有隐私政策
- [ ] 隐私政策是否包含必要条款
- [ ] 是否遵循最小必要原则
- [ ] 敏感信息是否取得单独同意
- [ ] 是否提供用户权利（访问、更正、删除）
- [ ] 第三方SDK是否披露
- [ ] 数据出境是否合规

### 安全合规必查项
- [ ] 是否全站HTTPS
- [ ] 密码是否强哈希
- [ ] 是否有访问控制
- [ ] 是否有日志审计（≥6个月）
- [ ] 是否有漏洞管理
- [ ] 是否有应急响应预案
