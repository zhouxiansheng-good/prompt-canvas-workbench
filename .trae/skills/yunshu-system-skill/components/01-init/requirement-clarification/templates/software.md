# 程序/软件开发领域需求澄清模板

> 软件开发是技术实现，需深挖类型定位、技术约束、数据流、非功能需求。

## 领域专属问题清单

| # | 问题 | 选项示例 | 优先级 | 隐性需求挖掘点 |
|---|------|----------|--------|---------------|
| S1 | 程序类型？ | A) Web后端/API B) 桌面端应用 C) 移动端APP D) 小程序 E) 命令行工具 F) 算法/脚本 | 必须 | 影响技术栈、部署方式、开发周期 |
| S2 | 技术栈约束？ | A) 无约束，你推荐 B) 必须用XX语言/框架 C) 需兼容现有系统 D) 团队只会XX | 必须 | 用户说"你定"时仍需确认维护能力 |
| S3 | 核心功能模块？ | A) 单功能工具 B) 多模块系统 C) 微服务架构 D) 对接现有系统 | 必须 | 需拆解为可验收的子功能 |
| S4 | 数据存储？ | A) 无持久化存储 B) 本地文件/JSON C) 关系型数据库 D) NoSQL E) 需迁移现有数据 | 重要 | 影响数据模型设计、迁移方案 |
| S5 | 接口/集成需求？ | A) 无外部接口 B) 调用第三方API C) 提供API给外部 D) 双向集成 | 重要 | 影响接口设计、认证方案、限流策略 |
| S6 | 用户体系？ | A) 无用户概念 B) 简单本地用户 C) 多用户权限系统 D) 对接SSO/企业账号 | 重要 | 影响认证授权、数据隔离 |
| S7 | 性能要求？ | A) 无特殊要求 B) 响应时间<100ms C) 支持并发用户<100 D) 高并发/分布式 | 可选 | 用户常忽略，但影响架构设计 |
| S8 | 安全要求？ | A) 内部使用，无特殊要求 B) 基础安全（输入验证/防SQL注入）C) 需加密/审计日志 D) 等保/合规要求 | 可选 | 涉及敏感数据时必须确认 |
| S9 | 部署/运维？ | A) 本地运行即可 B) 需Docker容器化 C) 需CI/CD流水线 D) 需监控/告警 | 可选 | 影响DevOps工作量 |

## 常见问题模式

### 模式1："做个简单的工具"
- 用户常这么说："就做个简单的小工具，很快吧？"
- 实际可能想要：需要GUI、需要跨平台、需要打包分发
- 建议追问："这个工具给谁用？他们怎么安装和运行？"

### 模式2：技术栈"你定"
- 用户常这么说："技术栈你定，我不懂"
- 实际风险：选了他团队维护不了的技术
- 建议追问："后期谁维护？团队熟悉什么语言？"

### 模式3：忽略数据量和并发
- 用户常这么说："就几个人用，数据不多"
- 实际风险：上线后用户增长，架构无法扩展
- 建议追问："如果用户量增长10倍，现在的方案能撑住吗？需要预留扩展点吗？"

### 模式4：边界不清
- 用户常这么说："帮我优化一下现有系统"
- 实际风险：优化范围不清，改动影响未知
- 建议追问："优化哪些模块？不动哪些模块？有测试覆盖吗？"

## 领域术语表

| 术语 | 含义 | 澄清要点 |
|------|------|----------|
| API | 应用程序接口，系统间通信契约 | 确认REST/GraphQL/gRPC |
| 微服务 | 将系统拆分为独立部署的小服务 | 确认是否真的需要，避免过度设计 |
| 数据库迁移 | 数据结构变更时的数据转换 | 确认现有数据量和停机窗口 |
| 并发 | 同时处理多个请求的能力 | 确认峰值QPS |
| 异步 | 非阻塞式处理，如消息队列 | 确认是否需要实时反馈 |
| 幂等 | 同一操作多次执行结果相同 | 涉及支付/写入时必须确认 |
| 回滚 | 发布失败后恢复到上一版本 | 确认是否有备份和回滚方案 |

## 验收标准模板

```yaml
done_definition:
  - "核心功能模块可正常运行，通过单元测试"
  - "[如有接口] API文档完整，接口可调用"
  - "[如有数据库] 数据模型设计合理，可CRUD"
  - "[如有用户] 注册/登录/权限流程完整"
  - "代码通过基础质量检查（无严重Lint错误）"
  - "[如需要] 可部署到目标环境并运行"
  - "[如需要] 基础监控/日志可查看"
```

## 风险提示

| 风险 | 说明 | 建议 |
|------|------|------|
| 技术债务 | 用户要求"快"而牺牲质量 | 明确告知短期快 vs 长期维护的权衡 |
| 第三方依赖 | 依赖外部API/库，可能变更 | 确认版本锁定和降级方案 |
| 数据丢失 | 无备份机制 | 确认数据备份策略 |
| 权限提升 | 安全漏洞导致未授权访问 | 涉及敏感操作时必须做权限检查 |
