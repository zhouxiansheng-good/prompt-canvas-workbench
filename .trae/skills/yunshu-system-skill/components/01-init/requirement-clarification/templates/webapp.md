## 常见问题模式

### 模式1："做个简单的官网"
- 用户常这么说："就做个简单的公司官网，几页就行"
- 实际可能想要：需要CMS后台、多语言、联系表单、甚至在线客服
- 建议追问："官网内容谁来更新？需要客户留言/咨询功能吗？"

### 模式2："要好看，要大气"
- 用户常这么说："设计要好看，要大气"
- 实际风险：主观审美无标准，反复修改
- 建议追问："有喜欢的参考网站吗？或者提供品牌色/Logo？"

### 模式3：忽略移动端
- 用户常这么说："主要是PC看，手机无所谓"
- 实际风险：国内移动端流量占比超60%
- 建议追问："你的用户主要用什么设备访问？我可以查一下流量数据"

### 模式4："先上线再优化"
- 用户常这么说："先做出来上线，性能后面再优化"
- 实际风险：首屏加载慢直接影响跳出率和SEO
- 建议追问："目标用户网络环境如何？如果3秒打不开会流失多少？"

## 领域术语表

| 术语 | 含义 | 澄清要点 |
|------|------|----------|
| SSR | 服务端渲染，首屏由服务器生成HTML | SEO友好，需服务器 |
| SSG | 静态站点生成，构建时生成HTML | 速度快，适合内容不频繁更新 |
| CSR | 客户端渲染，浏览器执行JS生成页面 | 交互强，首屏慢 |
| 响应式 | 一套代码适配多终端 | 确认断点范围 |
| 首屏时间 | 用户看到有意义内容的时间 | 影响跳出率 |
| CMS | 内容管理系统，非技术用户可更新内容 | 确认谁负责更新 |
| 落地页 | 营销专用单页，引导转化 | 确认转化目标 |

## 验收标准模板

```yaml
done_definition:
  - "所有页面在目标终端正常显示，无布局错乱"
  - "导航/链接可正常跳转，无404"
  - "表单可提交，数据可接收"
  - "[如需要] CMS后台可增删改内容"
  - "[如需要] 基础SEO配置完整（标题/描述/结构化数据）"
  - "[如需要] 多语言切换正常"
  - "部署上线，可通过域名访问"
```

## 风险提示

| 风险 | 说明 | 建议 |
|------|------|------|
| 设计反复 | 无参考/无品牌规范导致主观修改 | 要求提供参考网站或情绪板 |
| 浏览器兼容 | 用户可能使用旧版浏览器 | 确认目标浏览器版本 |
| 内容缺失 | 开发时无真实内容，用占位符 | 确认内容提供时间 |
| 第三方服务 | 地图/统计/客服等嵌入 | 确认账号和API Key |
