# 执行沙箱规范

> 定义软件命令的安全执行环境，防止误操作和系统损坏。

## 沙箱原则

**"高风险必停"** — 任何可能破坏数据的操作都必须经过确认。

## 执行前检查清单

| 检查项 | 说明 | 不通过的处理 |
|--------|------|-------------|
| **软件存在性** | 目标软件是否已安装？ | 报告错误，提供安装指导 |
| **版本兼容性** | 软件版本是否在支持范围内？ | 报告警告，建议升级/降级 |
| **权限检查** | 是否有执行该命令的权限？ | 请求提升权限或更换操作 |
| **路径合法性** | 文件路径是否在允许范围内？ | 拒绝执行，报告非法路径 |
| **命令合法性** | 命令是否在白名单内？ | 拒绝执行，报告非法命令 |
| **数据备份** | 是否会修改现有数据？ | 提示备份，用户确认后执行 |

## 执行流程

```
Step 1: 用户确认
  展示将要执行的命令序列
  说明每个命令的作用和风险
  请求用户确认（使用当前平台 adapter 指定的原生交互工具）
  
Step 2: 预执行检查
  检查软件是否运行
  检查目标文件/目录是否存在
  检查磁盘空间是否充足
  
Step 3: 执行命令
  按顺序执行命令
  捕获 stdout 和 stderr
  记录执行时间
  
Step 4: 结果验证
  根据命令定义的 verification 方法验证
  检查退出码
  检查输出是否包含预期内容
  
Step 5: 错误处理
  如果失败 → 记录错误 → 尝试回滚 → 报告用户
  如果成功 → 记录结果 → 继续下一个命令
```

## 安全约束

| 约束 | 限制 | 说明 |
|------|------|------|
| 单命令超时 | ≤60秒 | 防止长时间挂起 |
| 总执行超时 | ≤5分钟 | 防止批量命令执行过久 |
| 文件操作范围 | 仅限工作目录 | 防止误删系统文件 |
| 网络访问 | 仅限localhost | 防止外部攻击 |
| 资源限制 | CPU≤50%, 内存≤1GB | 防止资源耗尽 |

## 回滚机制

```
执行前记录状态:
  - 备份将被修改的文件
  - 记录当前软件状态
  - 记录当前工作目录内容

执行失败时回滚:
  - 恢复备份的文件
  - 撤销已执行的命令（如果支持）
  - 清理临时文件

回滚验证:
  - 确认文件已恢复
  - 确认软件状态正常
```

## 与用户交互

```
执行前展示:
  "即将执行以下命令："
  "1. CreateProject --name Project1 --type municipal"
  "   作用：创建市政道路项目"
  "   风险：会生成新文件，不会修改现有文件"
  
  "是否继续？"
  [确认] [取消] [查看详细参数]

执行后展示:
  "命令执行结果："
  "✅ CreateProject → 成功（2.3s）"
  "   生成文件：Project1.hyprj"
  "   项目路径：C:\Projects\Project1"
```
