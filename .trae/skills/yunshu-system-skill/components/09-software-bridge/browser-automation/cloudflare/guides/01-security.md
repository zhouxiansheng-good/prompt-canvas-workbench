# Cloudflare Pages 部署 — 安全指南

---

## 1. 账号安全

- **会话保持**：Cloudflare Dashboard 依赖浏览器 Cookie 维持登录状态，不要在公共或共享设备上使用自动化
- **账号 ID 保护**：`CLOUDFLARE_ACCOUNT_ID` 虽非敏感凭证，但属于账号标识信息，避免公开分享
- **API Token 与浏览器分离**：本技能仅使用浏览器自动化，不涉及 API Token。如需 API 操作，使用独立的 Cloudflare API Token 并遵循最小权限原则

## 2. 部署权限

- **项目级隔离**：确保自动化操作的项目是目标项目，避免误部署到其他项目
- **生产环境确认**：部署到生产分支前，二次确认目标项目和源文件夹
- **回滚准备**：保留上一次成功部署的链接，以便快速回滚

## 3. 文件上传安全

- **路径校验**：`DEPLOY_SOURCE_PATH` 必须指向预期的构建输出目录，禁止上传源码目录或系统目录
- **敏感文件排查**：上传前确认 `dist` 中不包含 `.env`、密钥文件、源码映射（source map）等敏感内容
- **文件大小监控**：异常大的上传可能包含未预期的文件，应触发人工确认

## 4. 环境变量管理

- **本地存储**：`.env` 文件仅保存在本地，禁止提交到版本控制
- **最小配置**：仅配置必要的变量，不存储密码或 API 密钥
- **路径转义**：Windows 路径中的反斜杠需正确处理，避免转义问题

## 5. 生产环境注意事项

- **自定义域名**：若项目绑定了自定义域名，部署成功后自动生效，确保在测试环境验证后再部署生产
- **构建命令**：本技能为直接上传静态文件，不涉及 Cloudflare Pages 的 CI 构建。若项目依赖构建步骤，确保本地构建完成后再执行上传
- **缓存策略**：部署后静态资源可能受 CDN 缓存影响，关键更新后需确认缓存刷新策略
