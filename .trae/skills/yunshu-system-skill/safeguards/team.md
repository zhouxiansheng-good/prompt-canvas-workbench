## 回归测试检查清单（P-070）

| 检查项 | 检查方式 | 必要性 |
|--------|----------|--------|
| 修改前测试全绿 | 运行全量测试 | 必须 |
| 修改后测试全绿 | 运行全量测试 | 必须 |
| 新增测试覆盖变更 | 检查测试覆盖率 | 必须 |
| 关联功能回归测试 | 运行受影响模块的测试 | 必须 |
| 边界条件回归 | 运行边界测试 | 推荐 |
| 性能回归 | 运行性能基准测试 | 推荐 |
| 安全回归 | 运行安全扫描 | 推荐 |

## 回归防护流程

```
修改代码前：
  ↓
1. 运行全量测试，确认基线全绿
  ↓
2. 记录测试基线（通过数/失败数/跳过数）
  ↓
修改代码后：
  ↓
3. 运行全量测试
  ↓
4. 对比基线：
   ├─ 新增失败 → 回归！必须修复后才能继续
   ├─ 全绿但覆盖率下降 → 补充测试
   └─ 全绿且覆盖率不降 → 通过
  ↓
5. 记录回归测试结果到变更日志
```

## 权宜之计补丁检测（P-071）

| 补丁信号 | 检测方式 | 处理方式 |
|----------|----------|----------|
| TODO/FIXME/HACK注释 | 搜索代码 | 记录技术债务 |
| 硬编码值 | 搜索魔法数字 | 标记为待重构 |
| 临时绕过逻辑 | 搜索workaround/bypass | 标记为待修复 |
| 注释掉的代码 | 搜索注释块 | 删除或记录原因 |
| 过度复杂的条件 | 圈复杂度检查 | 标记为待简化 |

## 补丁管控规则

| 规则 | 说明 | 违反处理 |
|------|------|----------|
| 每个HACK必须有对应Issue | 技术债务必须跟踪 | 不许提交 |
| 补丁存活不超过2周 | 临时方案不能变永久 | 到期提醒重构 |
| 同一问题不许打3次补丁 | 3次补丁=需要架构重构 | 触发架构评审 |
| 补丁必须标注过期时间 | `// HACK: 2026-05-20前修复` | 不许提交无期限补丁 |

- 🚫 同一问题打3次以上补丁（P-071需要架构重构）
