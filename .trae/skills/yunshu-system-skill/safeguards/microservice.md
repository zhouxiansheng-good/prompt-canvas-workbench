## AI常见微服务错误

| 错误模式 | 具体表现 | 正确做法 |
|----------|----------|----------|
| 服务间同步调用链过长 | A→B→C→D串行调用，任一失败全链路失败 | 使用异步消息+事件驱动 |
| 无熔断器 | 下游服务故障时持续重试，拖垮上游 | 添加熔断器（Circuit Breaker） |
| 无服务发现 | 硬编码服务地址 | 使用服务注册/发现机制 |
| 无分布式追踪 | 请求跨服务后无法追踪全链路 | 添加Trace ID + 分布式追踪 |
| 共享数据库 | 多个服务共享同一数据库 | 每个服务独立数据库 |
| 无限流降级 | 流量突增时服务直接崩溃 | 限流+降级策略 |

## 检查规则

1. **服务边界**：每个服务是否有独立的数据存储？
2. **通信方式**：服务间通信是否合理（同步vs异步）？
3. **容错机制**：是否有熔断器/超时/重试策略？
4. **可观测性**：是否有日志聚合+分布式追踪+指标监控？
5. **数据一致性**：跨服务事务是否使用Saga模式？

---

## 反模式

- 🚫 多服务共享数据库（耦合）
- 🚫 服务间同步调用链过长（级联失败）
- 🚫 无熔断器（故障扩散）
- 🚫 无分布式追踪（无法定位问题）
- 🚫 硬编码服务地址（无法弹性伸缩）
