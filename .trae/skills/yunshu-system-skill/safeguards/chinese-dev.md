> 加载方式：由03-execute和01-init组件检测到中文开发者场景时触发读取本文件

## 第一层：中文编码防护（P-038）

> 在03-execute阶段触发，确保代码正确处理中文编码。

### 编码规范

| 场景 | 规范 | 说明 |
|------|------|------|
| 文件编码 | 统一使用UTF-8（无BOM） | 所有源码文件必须UTF-8 |
| HTML页面 | `<meta charset="UTF-8">` | 必须声明编码 |
| 数据库连接 | `charset=utf8mb4` | MySQL必须用utf8mb4支持emoji |
| HTTP响应 | `Content-Type: text/html; charset=utf-8` | 必须声明编码 |
| API请求 | `Accept-Charset: utf-8` | 请求头声明编码 |
| 配置文件 | UTF-8 with BOM 或 无BOM | 按项目约定统一 |

### 中文编码检查清单

| 检查项 | 检查方式 | 常见问题 |
|--------|----------|----------|
| 文件编码声明 | 检查编辑器配置/.editorconfig | 缺少编码声明 |
| 字符串处理 | 检查字符串截取/排序/正则 | 中文字符被截断 |
| 输入验证 | 检查表单验证/数据校验 | 中文输入被拒绝 |
| 文件读写 | 检查文件操作是否指定编码 | 读写乱码 |
| 日志输出 | 检查日志是否支持中文 | 日志乱码 |
| 邮件/通知 | 检查邮件编码设置 | 邮件乱码 |

### 中文变量名/注释规则

| 场景 | 规则 | 说明 |
|------|------|------|
| 变量名 | 不推荐中文变量名 | 部分工具链不支持 |
| 函数名 | 不推荐中文函数名 | 部分工具链不支持 |
| 注释 | 推荐中文注释 | 提高可读性 |
| 文档 | 推荐中文文档 | 面向中文开发者 |
| commit message | 中英文均可 | 按团队约定统一 |
| README | 推荐中英双语 | 扩大受众 |

## 第二层：中文文档适配（P-037）

> 在03-execute阶段触发，确保AI生成的文档符合中文开发者习惯。

### 中文文档规范

| 文档类型 | 规范 | 说明 |
|----------|------|------|
| README | 标题+简介+安装+使用+配置+常见问题 | 中文开发者习惯看完整的使用说明 |
| API文档 | 请求示例+响应示例+错误码+注意事项 | 必须有完整的中文示例 |
| 代码注释 | 关键逻辑必须注释，注释用中文 | "为什么"比"是什么"更重要 |
| 变更日志 | 版本号+日期+变更内容+影响范围 | 中文开发者关注兼容性影响 |
| 配置说明 | 每个配置项+类型+默认值+说明 | 缺一不可 |

### AI文档生成中文适配

| 生成场景 | 适配要求 | 常见AI错误 |
|----------|----------|-----------|
| 生成README | 必须包含中文安装说明 | 只写英文安装命令 |
| 生成API文档 | 必须有中文请求/响应示例 | 只有英文示例 |
| 生成注释 | 关键逻辑用中文注释 | 只写英文注释或无注释 |
| 生成错误信息 | 用户可见的错误信息用中文 | 只有英文错误信息 |
| 生成配置说明 | 配置项说明用中文 | 只有英文说明 |

### 中文技术术语规范

| 英文术语 | 推荐中文 | 不推荐 |
|----------|----------|--------|
| deploy | 部署 | 发布（易混淆） |
| repository | 仓库 | 存储库 |
| dependency | 依赖 | 依存 |
| middleware | 中间件 | — |
| environment variable | 环境变量 | — |
| configuration | 配置 | 设定 |
| cache | 缓存 | — |
| token | 令牌/Token | 标记 |

## 第三层：中文生态适配（P-039）

> 在01-init和03-execute阶段触发，确保代码适配国内技术生态。

### 国内生态适配清单

| 场景 | 适配项 | 说明 |
|------|--------|------|
| npm安装 | 使用国内镜像（npmmirror.com） | 加速安装 |
| pip安装 | 使用国内镜像（pypi.tuna.tsinghua.edu.cn） | 加速安装 |
| Docker镜像 | 使用国内镜像源 | 加速拉取 |
| GitHub访问 | 提供Gitee/镜像备选方案 | 网络不通时的替代 |
| CDN资源 | 使用国内CDN（bootcdn/unpkg镜像） | 加速加载 |
| API服务 | 考虑国内云服务（阿里云/腾讯云） | 合规+速度 |
| 短信/邮件 | 使用国内服务商 | 合规要求 |
| 支付 | 使用支付宝/微信支付 | 国内主流 |
| 地图 | 使用高德/百度地图 | 国内合规 |
| 数据存储 | 考虑国内云存储 | 合规要求 |

### 项目初始化中文适配（01-init触发）

```
项目初始化时检查：
  ↓
1. 确认目标用户是否包含中文用户
  ↓
2. 如果是，自动配置：
   ├─ .npmrc: registry=https://registry.npmmirror.com
   ├─ .editorconfig: charset=utf-8
   ├─ .gitattributes: * text=auto eol=lf
   ├─ README.md: 中英双语
   └─ CI/CD: 国内镜像源
  ↓
3. 提醒开发者注意：
   ├─ ICP备案要求（如果是网站）
   ├─ 数据存储合规（个人信息保护法）
   └─ 网络安全要求
```

### 部署环境适配

| 部署场景 | 适配项 | 说明 |
|----------|--------|------|
| 阿里云 | 函数计算/OSS/SLB | 国内最常用 |
| 腾讯云 | 云函数/COS/CLB | 国内常用 |
| 华为云 | FunctionGraph/OBS/ELB | 政企常用 |
| 私有部署 | Docker/K8s内网 | 企业内网 |

## 反模式

- 🚫 文件编码不统一导致乱码（P-038）
- 🚫 数据库用utf8而不是utf8mb4（P-038）
- 🚫 只写英文注释不写中文注释（P-037）
- 🚫 README只有英文没有中文（P-037）
- 🚫 npm安装不配置国内镜像（P-039）
- 🚫 不考虑GitHub访问问题（P-039）
- 🚫 用户可见的错误信息只有英文（P-037）
- 🚫 不考虑国内合规要求（P-039）
