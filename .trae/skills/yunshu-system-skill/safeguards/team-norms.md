# 团队AI使用规范

> 覆盖问题：P-072 影子AI失控 / P-073 代码审查瓶颈 / P-074 安全隐患普遍
> 触发场景：代码验收时（04-accept）
> 加载方式：04-accept债务与团队门禁触发

---

## 问题描述

| 编号 | 问题 | 严重程度 | 具体表现 |
|------|------|---------|---------|
| P-072 | 影子AI失控 | 🟠 严重 | 66.7%的团队面临开发者未经审批擅自使用AI工具 |
| P-073 | 代码审查瓶颈 | 🟠 严重 | AI生成速度是人类审查的10-100倍 |
| P-074 | 安全隐患普遍 | 🟠 严重 | 45%的AI生成代码存在安全隐患 |

---

## 影子AI管控（P-072）

| 管控维度 | 规范 | 说明 |
|----------|------|------|
| AI工具白名单 | 只允许团队批准的AI工具 | 防止未经审批的工具 |
| 代码来源标记 | 标记AI生成代码比例 | 可追溯 |
| 审批流程 | AI生成代码必须经过审查 | 不许直接合入主分支 |
| 使用日志 | 记录AI工具使用情况 | 可审计 |
| 安全评估 | 新AI工具引入前必须安全评估 | 防止数据泄露 |

## 代码审查加速策略（P-073）

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 分批审查 | 每批不超过5个文件 | 大量变更 |
| 优先级审查 | 安全相关>核心逻辑>辅助代码 | 时间有限 |
| 自动化前置 | lint/test/安全扫描先跑 | 所有场景 |
| 差异审查 | 只审查变更部分，不重读全文件 | 小改动 |
| 结对审查 | 两人一起审查AI代码 | 高风险变更 |

## 安全审查检查清单（P-074）

| 检查项 | 检查方式 | 严重度 |
|--------|----------|--------|
| 输入验证 | 检查所有用户输入是否验证 | 🔴 |
| 权限检查 | 检查敏感操作是否有权限控制 | 🔴 |
| SQL注入 | 检查数据库操作是否参数化 | 🔴 |
| XSS防护 | 检查输出是否转义 | 🔴 |
| 敏感数据 | 检查是否有硬编码密钥/密码 | 🔴 |
| 日志安全 | 检查日志是否泄露敏感信息 | 🟠 |
| 依赖安全 | 检查依赖是否有已知漏洞 | 🟠 |

## 反模式

- 🚫 使用未经审批的AI工具（P-072影子AI）
- 🚫 AI代码不审查直接合入（P-073审查瓶颈）
- 🚫 不做安全扫描就交付（P-074安全隐患）
