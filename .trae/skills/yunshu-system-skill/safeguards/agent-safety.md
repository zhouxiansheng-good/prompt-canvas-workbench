## Agent安全铁律

### 1. Tool Poisoning防护（P-043）

| 检查项 | 说明 |
|--------|------|
| 工具来源验证 | 只使用官方或已验证的工具 |
| 描述审查 | 审查工具描述是否有可疑指令（如"忽略之前指令"） |
| 权限最小化 | 工具只申请必要权限 |
| 行为监控 | 监控工具调用是否超出预期范围 |

### 2. Rug Pull防护（P-044）

| 检查项 | 说明 |
|--------|------|
| 远程工具锁定 | 授权后锁定工具版本，不接受动态更新 |
| 变更审计 | 工具行为变更时记录审计日志 |
| 权限定期审查 | 定期检查已授权工具的权限是否仍然必要 |

### 3. 信任链管理（P-045）

| 信任域 | 安全边界 |
|--------|----------|
| Client（用户端） | 用户意图是最高优先级 |
| Server（工具端） | 工具执行结果需验证 |
| LLM（模型端） | 模型输出需审查 |

**原则**：不盲目信任任何单一信任域的输出

### 4. 无限循环防护（P-046）

| 防护机制 | 说明 |
|----------|------|
| 调用次数上限 | 单次任务工具调用不超过N次（默认10次） |
| 重复检测 | 连续3次相同调用→强制停止 |
| 超时机制 | 单次工具调用超时→跳过 |
| 人工确认 | 超过5次调用→请求用户确认是否继续 |

### 5. 上下文腐败防护（P-047）

| 防护机制 | 说明 |
|----------|------|
| 关键信息标记 | 标记不可压缩的关键上下文 |
| 压缩后验证 | 压缩后验证关键决策是否保留 |
| 冗余存储 | 关键信息同时存在文件中（不只在上下文中） |

### 6. 成本控制（P-048）

| 控制机制 | 说明 |
|----------|------|
| Agent数量限制 | 同时运行的Agent不超过3个 |
| 单Agent token上限 | 每个Agent有token预算 |
| 总量预警 | 总token消耗达到阈值时预警 |

### 7. 非确定性应对（P-049）

| 应对策略 | 说明 |
|----------|------|
| 结果验证 | Agent输出必须通过验证步骤 |
| 幂等设计 | 相同输入应产生相同效果 |
| 重试机制 | 结果不正确时允许重试（有次数限制） |

## 反模式

- 🚫 盲目信任工具描述（Tool Poisoning）
- 🚫 不限制工具调用次数（无限循环）
- 🚫 不验证Agent输出（非确定性风险）
- 🚫 不设token预算（成本爆炸）
- 🚫 关键信息只存在上下文中（上下文腐败）
- 🚫 远程工具授权后不锁定版本（Rug Pull）
