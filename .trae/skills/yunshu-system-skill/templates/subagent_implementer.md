# 实现者子智能体分派模板

使用此模板分派实现者子智能体。子智能体已内置云舒微流程（澄清→计划→执行→自验收），此模板提供控制器侧的分派上下文。

```text
<当前平台的子智能体调用方式>
agent/subagent: "yunshu-implementer"
  description: "实现任务 N：[任务名]"
  query: |
    任务描述：[完整粘贴任务文本 — 不要让子智能体读文件，直接提供]

    上下文：[场景设定：此任务在整体中的位置、依赖关系、架构上下文、已有代码模式]

    工作目录：[目录路径]
```

平台映射：Trae 使用 Task tool / 智能体面板；Codex 只有用户明确授权子智能体时才使用 Codex subagent；其他平台没有等价能力时回退到 03-execute 串行执行。

**分派铁律**：
- 任务描述必须完整粘贴，不要让子智能体自己读文件
- 上下文必须包含架构、依赖、已有代码模式
- 工作目录路径必须正确
- 任务间的依赖关系必须明确

**处理实现者状态**：

| 状态 | 含义 | 处理方式 |
|------|------|---------|
| DONE | 完成，无异议 | 进入规范审查 |
| DONE_WITH_CONCERNS | 完成但有疑虑 | 读取疑虑：正确性/范围问题→先解决再审查；观察性意见→记录并继续审查 |
| NEEDS_CONTEXT | 缺少信息 | 提供缺失上下文，重新分派 |
| BLOCKED | 无法完成 | 评估阻塞：上下文问题→补充后重派；任务过大→拆分；计划有误→升级给用户 |
