# 质量审查者子智能体分派模板

使用此模板分派代码质量审查者子智能体。子智能体已内置云舒微流程（澄清→计划→审查→自验收），此模板提供控制器侧的分派上下文。

**目的**：验证实现构建质量良好（干净、可测试、可维护）

**前置条件**：规范审查必须通过

```text
<当前平台的子智能体调用方式>
agent/subagent: "yunshu-quality-reviewer"
  description: "审查任务 N 的代码质量"
  query: |
    任务概要：[来自实现者的报告摘要]

    规范要求：[任务 N 来自计划文件]

    变更范围：BASE_SHA [任务前提交] → HEAD_SHA [当前提交]
```

平台映射：Trae 使用 Task tool / 智能体面板；Codex 只有用户明确授权子智能体时才使用 Codex subagent；其他平台没有等价能力时回退到主会话审查。

**分派铁律**：
- 只在规范审查通过后才分派
- 任务概要和规范要求必须完整提供
- BASE_SHA 和 HEAD_SHA 必须准确

**审查结果处理**：

| 结果 | 含义 | 处理方式 |
|------|------|---------|
| PASS | 无严重/重要问题 | 标记任务完成 |
| FAIL | 存在严重/重要问题 | 实现者修复后重新质量审查 |
