# 规范审查者子智能体分派模板

使用此模板分派规范审查者子智能体。子智能体已内置云舒微流程（澄清→计划→审查→自验收），此模板提供控制器侧的分派上下文。

**目的**：验证实现者构建了规范要求的东西（不多不少）

**前置条件**：实现者必须返回 DONE 或 DONE_WITH_CONCERNS

```text
<当前平台的子智能体调用方式>
agent/subagent: "yunshu-spec-reviewer"
  description: "审查任务 N 的规范符合性"
  query: |
    规范要求：[完整粘贴任务规范]

    实现者报告：[来自实现者的报告]

    变更文件：[实现者修改的文件列表]
```

平台映射：Trae 使用 Task tool / 智能体面板；Codex 只有用户明确授权子智能体时才使用 Codex subagent；其他平台没有等价能力时回退到主会话审查。

**分派铁律**：
- 规范要求必须完整粘贴
- 实现者报告必须完整提供
- 变更文件列表必须准确
- 规范审查者发现的问题 = 未完成，不是"差不多"

**审查结果处理**：

| 结果 | 含义 | 处理方式 |
|------|------|---------|
| PASS | 规范符合 | 进入质量审查 |
| FAIL | 发现问题 | 实现者修复后重新规范审查 |
